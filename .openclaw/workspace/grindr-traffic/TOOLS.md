# TOOLS.md - 工具与环境说明（Grindr流量运营）

## 目标

记录本工作区可用工具、调用约定与注意事项，确保执行稳定。

## 上游协作

- 默认接收来自 `ops-chief` 的分派任务（分派前缀 `gr:`）。
- 任务应包含：背景、目标、约束、截止时间、验收标准。
- 输出需经过质检验收后才能交付给用户。

## 下游依赖

- **Grindr API 技能**：待封装（需要提供完整 API 文档、认证方式、请求示例）。
  - 获取附近用户列表（坐标、半径、筛选条件）
  - 获取用户资料（用户ID、字段列表）
  - 发送私信（接收者ID、消息内容、类型）
  - 关注用户（用户ID、操作类型）

## 工具使用约定

- **API调用**：封装为 Python 脚本，统一错误处理、重试、日志记录。
- **地理参数**：坐标格式（经纬度）、半径（米/公里）、时间窗口。
- **频率控制**：遵守平台速率限制，实现自动休眠与队列调度。
- **会话管理**：维护对话状态（新接触、已回复、已转化），避免重复打扰。
- **Nearby 返回收敛**：`grindr-discovery-reader/get_nearby_profiles.sh` 必须在本地执行 topN 裁剪后再输出，默认按 `limit`（缺省 5）收敛结果体积。
- **Nearby 交付优先**：拿到成功响应后立即回传前 N 条结果；若需排障，必须在交付后进行，禁止边交付边无关调试。
- **执行超时约束**：所有 `curl` 请求默认 `--max-time 20`；其他外部命令默认控制在 25 秒内并及时回传进度。
- **Nearby 固定调用链**：东京/城市附近用户任务必须优先走 `city-geocode` + `get_nearby_profiles.sh`，禁止临时新建调试脚本替代主链路。
- **Chat 固定调用链**：所有 WS 发送类任务优先使用 `skills/grindr-chat-manager/scripts/ws_send_and_pull.sh`，保证 `connect/send/pull` 三段原子执行。
- **Chat 结果返回**：默认回传完整 JSON，至少包含 `steps.connect`、`steps.send`、`steps.pull` 三段结果。

## Telegram 任务模板（chat-manager）

- 通用发送链路模板：
  - `请调用 grindr-chat-manager 的 ws_send_and_pull，payload=<JSON>，pullLimit=20，clear=true，forceReconnect=false，返回完整JSON。`
- 文本消息模板：
  - `请调用 grindr-chat-manager 的 message_send_text，sender=<ID>，target=<ID>，text=<文本>，返回完整JSON（包含connect/send/pull）。`

## 安全与风控

- **认证信息**：API 密钥、Token 等敏感信息通过环境变量传递，不硬编码。
- **隐私保护**：不持久化存储用户个人身份信息，仅保留运营所需元数据。
- **合规检查**：每次外发动作前确认是否符合平台用户协议与当地法规。
- **账号安全**：监控账号状态（登录异常、限制通知），及时预警。

## 故障模式与排障

- **认证失败**：检查 Token 有效期、权限范围、IP 限制。
- **无用户数据**：验证坐标正确性、半径设置、时间筛选条件。
- **消息发送失败**：确认接收者是否接受消息、频率限制、内容合规。
- **API 变更**：监控响应结构变化，及时更新封装脚本。

## 性能与扩展

- **批量处理**：支持坐标网格化扫描、用户列表分批次触达。
- **异步执行**：长时间任务（如大规模用户发现）支持后台运行与进度查询。
- **策略复用**：将成功策略保存为模板，支持一键复制到新坐标区域。
