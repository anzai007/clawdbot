# TOOLS.md - 工具与环境说明（运营总负责人）

## 目标

记录本工作区可用工具、调用约定与注意事项，确保执行稳定。

## 常用能力

- OpenClaw 会话与路由管理
- Telegram 账号路由（`ops` 账号）
- 与 `x-traffic` 的任务分派协作

## 协作工具约定

- 分派给 `x-traffic` 的任务，必须带完整上下文与交付标准。
- 分派给 `meta-ads` 的任务，同样需要完整上下文与交付标准。
- 分派给 `grindr-traffic` 的任务，同样需要完整上下文与交付标准。
- 对执行结果做验收后再输出给用户。
- **分派代号**：为简化输入，使用以下代号自动识别并分派任务：
  - **X/Twitter 任务 → `x-traffic`**：
    - 消息以 `xt:` 开头（如 `xt: 评论这条推文`）
    - 消息包含 `推` 字且上下文明确为 Twitter 任务（如 `发推`、`推一下`、`推特任务`）
    - 避免歧义：非 Twitter 相关语境中的“推”字不会触发。
  - **Meta Ads 任务 → `meta-ads`**：
    - 消息以 `meta:` 开头（如 `meta: 查看广告账户`）
    - 消息包含 `广告` 或 `meta` 字且上下文明确为 Meta Ads 任务（如 `广告账户`、`meta广告`）
  - **Grindr 任务 → `grindr-traffic`**：
    - 消息以 `gr:` 开头（如 `gr: 发现附近用户`）
    - 消息包含 `grindr`、`附近`、`坐标`、`地理位置` 字且上下文明确为 Grindr 任务（如 `grindr用户`、`附近的人`、`坐标筛选`）

## 社媒相关

- X/Twitter 执行优先交由 `x-traffic`。
- 涉及 TweetAPI 调用时，检查参数完整性：`TWEETAPI_KEY`、`authToken`、`proxy`。

## 维护规则

- 新增工具时在此登记“用途 + 调用限制 + 风险”。
- 发现失败模式时在此补充“排障步骤”。
